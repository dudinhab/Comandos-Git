Aprendendo a usar o GIT!

GIT é diferente do GitHub.

Git init

Cria um novo repositório para que seja gerada novas versões

Git add

adiciona arquivos na area de stage, que é um armazenamento antes do commit

Git commit

Grava as alterações que estão no stage e gera uma versão

Git status

Verifica o status atual do repositório, do que esta aguardando add e commit

Git log

Mostra todo o histórico de commit dado no projeto

Git checkout [numero do commit (ou) nome de uma branch]

Comando para navegação de entre commits e branchs

Git checkout -b [nome da nova branch]

Criação de uma nova branch

Git Branch

Visualização de todas as branchs criadas

Git Merger

Integração de Branch em uma master

*Branch | *Merge | *Master\Main | *Commit | *Stage
